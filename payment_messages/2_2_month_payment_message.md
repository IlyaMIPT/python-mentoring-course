Изучение SQL.
Для обучения SQL не выделяются отдельные спринты. Его мы изучаем параллельно с Pandas. Почему? SQL обрабатывает таблицы, Pandas тоже. Методы SQL близки к методам Pandas, и то, и то - инструменты обработки и анализа данных. Кроме того, изучение SQL - это всегда решение большого количества задач, которое вполне можно выполнить на специальном тренажере.

Варианты изучения SQL:
1). Самый быстрый и эффективный: приобрести мой курс SQL для аналитиков данных. К курсу идет тренажер с более чем 60 задачами. Задачи идут от простого к сложному, но без лишних наворотов. Кроме того, вы всегда можете у меня спросить, если что-то не получается или получить обратную связь по решению. В курсе представлены задачи адекватной сложности, которые помогают хорошо подготовиться к собеседованию на аналитика данных. Кроме того, курс предназначен именно для аналитиков, а не для представителей других ИТ-специальностей, что обуславливает наполнение курса, сложность задач и набор тем. Курс прошли десятки учеников и остались довольны. Ссылка на курс: https://клондайк-аналитика.рф/course/sql_basics
2). Изучать SQL по открытым источникам в интернете. В открытом доступе есть тренажеры SQL, есть даже бесплатные. Об их качестве судить не могу. Но необходимо четко понимать, что бесплатные тренажеры и курсы SQL для привлечения широкой аудитории вынуждены снижать сложность задач и материала. Иначе их просто не будут смотреть и площадка, на которой курс размещен, будет меньше этот курс показывать. В итоге авторы вынуждены давать в своих курсах и тренажерах элементарные задачи, которые ничему не учат. В противном случае, обычный пользователь, ни в чем не разобравшись, просто закроет тренажер и поставит низкую оценку курсу. Так работают массовые площадки курсов, их главная метрика - время, проведенное пользователем на сайте или в приложении. Таким образом, такие курсы годятся лишь для ознакомления с темой и непригодны для подготовки к реальным собеседованиям. На собеседованиях дают сложные задачи по SQL, т.к. простые все уже умеют решать. Отмечу, не называя имен, что есть также тренажеры SQL с гипертрофированной сложностью, содержащие, по сути, олимпиадные задачи. Это, в свою очередь, является вредным для начинающих - в SQL важно постепенное нарастание сложности.
В итоге не настаиваю на выборе вариантов, достоинства и риски я обозначил. SQL - исключительно важная тема, на каждом собеседовании на аналитика вы будете получать задачи и если не справитесь, оно будет провалено и путь в эту компанию будет закрыт. 
К счастью, SQL вполне по силам изучить с помощью курсов и тренажеров без помощи преподавателя. Хотел бы отметить, что если вы предпочтете бесплатные альтернативы в интернете или другие площадки (вариант 2), то ответственность за изучение и/или провал на собеседованиях по SQL вы берете на себя.
